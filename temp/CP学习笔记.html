标准输入输出流

显然， << 和 >> 操作符在此程序中的返回值是左边的 cout 和 cin，否则，不可能将所有要输出的东西都连接起来。<< 和 >> 可以用来连接不同的操作数，有整形，字符串等等，它们都可以正常工作，因为标准库里内置了不同版本的这些操作符，它们可以自己处理不同的类型。

  • std::cout 表示使用标准输入输出流库（就是 include 的尖括号写的 iostream）的 cout。
  • std 表示命名空间，用来解决命名冲突的，例如不同的库，有名称相同的类，或者自己写的与库中的冲突了。
  • :: 是域操作符，配合命名空间使用。
  • endl 表示一行的结束，并且会刷新设备的换冲区，自带换行效果。

C++ 算术类型

c++ 只规定了最小尺寸，并允许编译器使用更大的尺寸。

类型 含义 最小尺寸（bit）
bool 布尔 未定义
char 字符 8
wchar_t 宽字符 16
char16_t Unicode 字符 16
char32_t Unicode 字符 32
short 短整型 16
int 整型 16
long 长整型 32
long long 长整型 64
float 单精度浮点型 6位有效数字
double 双精度浮点型 10位有效数字
long double 扩展精度浮点型 10位有效数字

除去 bool 和 扩展字符型之外（bool 和 char 都属于整型），整型可以分为 unsigned 和 signed。

但是字符分为3种，unsigned char 和 signed char 和 char，注意 signed char 和 char 并不相同。尽管字符型有3中，但是其表现形式只有两种：带符号的和不带符号的。char 会表现为 char 或者 signed char，具体是那种由编译器决定。

类型选择：

  • 当值不可能为负数的时候，选择 unsigned 类型。
  • 整数运算时，使用int，通常 short 太小了，long 一般和 int 一样大，所以当要表示的范围超过 int 最大值，选择 long long。
  • 在进行算数运算的时候，不要使用 bool 和 char。因为不同的机器，对 char 有不同的实现，有的有符号有的无符号。当你要使用小范围的整数时，显示的指定 signed char 和 unsigned char。
  • 用double来进行浮点型运算，float通常精度太低，并且消耗的性能和 double 差不多，甚至在某些机器上还要差点。long double 一般没必要，并且消耗性能太高，一般用于专业硬件上。

类型转换

  1. 算数类型给bool 赋值时，非 0 值为true，否则为false。

  2. bool 赋值给其他算数类型，true 是 1， false 是 0；

  3. 浮点数赋值给整型时，取整，去掉小数点。

  4. 整型给浮点数赋值时，小数点后面为0，注意有可能丢失精度。

  5. 给一个无符号类型赋值，将赋值对表示的范围取模。取模公式： a % b = a - (a / b) * b。

    例如： -1 % 256 = -1 - (-1 / 256）* 256 = -1 - 0 = -1;

  6. 给一个有符号的类型赋值，当值超过其能够表示的范围，则赋值后的值是未定义的，程序可能崩溃，或者产生垃圾值。

编写程序的时候，避免使用未定义的行为，和依赖运行环境的实现。未定义的行为上面举过例子了，依赖于环境比如说，将 int 占得字节数看成固定的，这样写出来的程序就是不可移植的，在你的电脑上ok，在别的电脑上就无法运行了。

无符号类型之间的转换：

当在表达式中有无符号的有符号类型时，会都将其转换为无符号类型。

同时，还要注意无符号数相减的问题，这两者相减，不会产生小于0的数。

转义字符

  • 常见的转义字符

  • \x1234 与 \1234

    \x1234： 123 是16进值，代表字符集的一个字符。无论右面跟着多少数字，都代表一个字符。所以有的写法不合法。

    \1234： 123 是 8 进制，这种写法，后面最多只能有3个数字，其余的数字会当成单个的字符，比如 4 就会被当成一个字符。

引用

引用是别名，它本身不是一个对象，只是为一个已经存在的对象起一个另外的名字。因为不是对象，所以不能定义引用的引用。引用定义之后，无法再重新绑定好别的对象，所以定义的时候必须初始化。

定义一个引用之后，对引用的所有操作都作用在其绑定的对象上。获取引用的值，也是获取绑定对象的值。还有其他注意的地方，等到后面在做笔记。
