为Python程序指定解释器

大部分python文件的头部都会写上#!/usr/bin/python 或者 #!/usr/bin/env，这个语句主要和运行模式有关。

如果我们用普通运行模式，例如(linux) ：

那么上面的两种写法并没有区别，但是如果想让python程序像普通程序一样运行，例如：

这个语句就起作用了，他用来为脚本语言指定解释器，通常认为用 #!/usr/bin/env python 要比 #!/usr/bin/python 更好，因为 python 解释器有时并不安装在默认路径。

使用#!/usr/bin/env python写法在你机器上安装了多个版本的python的时候有意义，这样声明的时候，会去取你机器的 PATH 中指定的第一个 python 来执行你的脚本。

!/usr/bin/python表示写死了就是要 /usr/bin/python 这个目录下 python 来执行你的脚本。这样写程序的可移植性就差了，如果此路径下python命令不存在就会报错。

导入文件

通常一个具有完善功能的程序会有多个文件构成，每个文件都有各自的功能，通过导入就可以使用别的文件已经写好的功能。

导入有3中选择：

  • 自己写的
  • 别人写的
  • Python内置的

上面的sys就是python自带的。

假如我们编写了两个.py文件：index.py 和 login.py，index.py 中导入了 login.py，那么运行程序的时候会生成一个 login.pyc 文件，这个.pyc 文件就是Python解释器编译之后产生的字节码。每个更改源文件都会重新生成一份字节码，没有更改解释器会直接使用上次编译的源文件，优化程序执行速度。
