先说我们最熟悉的 Activity，Activity给人的感觉就是App的一个界面，这个界面由很多 View 组成，我们在Activity里做的操作都是对View的操作。所以给人的感觉就是 View 放在了 Activity 上面。

但是深入过源码的都知道，我们在Activity里面调用 setContentView 的时候，实际上是调用的 Window 的 setContentView 方法。那么从源码的角度来说，View 是放在了 Window 上面。

既然 View 是放在了 Window 上面，那么我就问一个问题：我们自己创建一个 Window 的时候，想要往 Window 里面添加 View 是需要使用 WindowManager 的，如果我们想要更新 View 也是需要用到 WindowManager 的，那么为什么我们能直接在 Activity 里对 View 进行更新？为什么我们直接调用 setText 就能设置文字？

其实这个是与 ViewRootImpl 有关，我们知道 ViewRootImpl 是专门绘制 View 的，它的 scheduleTraversals 方法里面就有利用 mWindowSession 来调用 WMS 的方法，从而更新 View，这是 IPC 调用。

网上有很多说话，最多的就是：Activity像一个工匠（控制单元），Window像窗户（承载模型），View像窗花（显示视图）。

其实用我自己的话来说就是：

Activity 最重要的是它提供了生命周期，有了生命周期，我们就可以更方便的来管理界面（Window 和 View）和资源。它确实是起着控制的作用，在 IOS 里面有另外一个名字 Controller。

View 要想显示出来，必须要贴在 Window 上，不然是不能显示的，但是 Window 要显示东西，不一定要靠 View。
