HashMap 源码分析

它根据键的hashCode值存储数据，大多数情况下可以直接定位到它的值，因而具有很快的访问速度，但遍历顺序却是不确定的。 HashMap最多只允许一条记录的键为null，允许多条记录的值为null。HashMap非线程安全，即任一时刻有多个线程同时写HashMap，可能会导致数据的不一致。如果需要满足线程安全，可以用 Collections的synchronizedMap方法使HashMap具有线程安全的能力，或者使用ConcurrentHashMap。

HashMap 的线程安全性，在多线程中，应该不使用 HashMap，而是使用线程安全的 ConcurrentHashMap。

网上有个例子：开两个线程，往同一个 HashMap 里面存放数据，当 HashMap 扩容的时候，会造成循环引用，是因为扩容时要拷贝链表。我看了下其实不用分析那么深，刚开始往里面放数据的时候，HashMap 的 size 有段时间是不对的，当然这个例子很有意思。

总结：

(1) 扩容是一个特别耗性能的操作，所以当程序员在使用HashMap的时候，估算map的大小，初始化的时候给一个大致的数值，避免map进行频繁的扩容。

(2) 负载因子是可以修改的，也可以大于1，但是建议不要轻易修改，除非情况非常特殊。

(3) HashMap是线程不安全的，不要在并发的环境中同时操作HashMap，建议使用ConcurrentHashMap。

(4) JDK1.8引入红黑树大程度优化了HashMap的性能。
