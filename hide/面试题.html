  3. 弱引用和软引用的区别，有没有用到过？

    强引用是使用最普遍的引用。如果一个对象具有强引用，那垃圾回收器绝不会回收它。

    如果一个对象只具有软引用，则内存空间足够，垃圾回收器就不会回收它；如果内存空间不足了，就会回收这些对象的内存。

    在垃圾回收器线程扫描它所管辖的内存区域的过程中，一旦发现了只具有弱引用的对象，不管当前内存空间足够与否，都会回收它的内存。

    虚引用主要用来跟踪对象被垃圾回收器回收的活动。虚引用与软引用和弱引用的一个区别在于：虚引用必须和引用队列 （ReferenceQueue）联合使用。当垃圾回收器准备回收一个对象时，如果发现它还有虚引用，就会在回收对象的内存之前，把这个虚引用加入到与之 关联的引用队列中。

    注意：软引用，弱引用，虚引用都可以和引用队列一起使用，只不过虚引用是强制的。

    http://www.jianshu.com/p/371e269ac7ae

    PS：弱引用 + 引用队列 + 手动触发GC 就是 LeakCanary 的原理。

    PPS：LeakCanary 什么时候触发内存泄露的检测呢？

    在消息队列为空的时候。

  4. 如何用一个数组实现一个队列？如果满了怎么办（扩容），扩容怎么实现？

    队列是一种特殊的线性表，特殊之处在于它只允许在表的前端（front）进行删除操作，而在表的后端（rear）进行插入操作，和栈一样，队列是一种操作受限制的线性表。进行插入操作的端称为队尾，进行删除操作的端称为队头。

    重点在于，必须要保存队头和队尾的索引位置。

    删除元素的时候，并不是真正的删除该元素，只是将指针向后移动。

    每次删除元素的时候，还要考虑是否需要移动元素，如果移动元素，则队头的位置不变，如果不移动元素，那么可以做一个循环数组。

    对于循环数组的实现，怎么实现扩容？

    既然要扩容，那么隐含条件就是，队列已经满了，就只需要分两次copy，第一次是从 head 到数组末尾，

    第二次就是从数组头部到 tail。使用System.arraycopy().

  5. java异常介绍

    在 Java 中，所有的异常都有一个共同的祖先 Throwable（可抛出）。

    类层次结构：

    Error（错误）:是程序无法处理的错误，表示运行应用程序中较严重问题。

    Exception（异常）:是程序本身可以处理的异常。

    比如说，我们需要解析json，将字符串转化为数字，我们就需要去try一下（试图执行某段逻辑），如果转换成功，则执行对的逻辑，如果转换失败执行错误处理的逻辑。

    通常用到的都是 try catch 语句，如果涉及到资源的关闭，还需要使用 finally。

    比如在volley的源码中，请求网络重试的逻辑，就使用到了异常，就是在一个while true循环内部，内部的代码用 try 包含起来，如果请求遇到了错误，就会抛出一个自定义的异常，在catch中去处理，在catch中判断请求次数。

  7. handler和looper之间的关系，他们怎样通信的，怎么实现事件的分发？

    在哪个线程创建的handler就会对应到哪个线程的looper，每个线程只有一个looper。handler 是通过 threadlocal 来获取 looper 对象，然后将msg方入队列中。looper里面实现了一个循环，从队列里面去取msg，取方法是阻塞的，取到之后，根据msg拿到handler，调用handler的处理方法。

  10. 链表插入节点，反转链表

    http://blog.csdn.net/feliciafay/article/details/6841115

    插入节点：假设想要在 Node A 之后插入 Node C，Node A 之后的节点是 Node B。首先获取到 B，将 C 的next 赋值为 B，然后将 A 的next 赋值为 C，这样就搞定了。操作不能反了。如果先将 A 的next赋值为C，那么就找不到B了。

    反转链表：我说一种非递归的方式。

    如果链表长度小于3，那么直接反转就好了，如果长度大于等于3，可以按照这样的一种思路：从第3个节点到第N个节点，依次逐节点插入到第1个节点(head节点)之后，(N-1)次这样的操作结束之后将第1个节点挪到新表的表尾即可。

    举例：将第3个节点，插入到第1个节点之后，有两布操作，首先将第3个节点拿出来，就是将第2个节点的next赋值为第3个节点的next。然后将第3个节点插入到第一个节点之后，这和上面的插入操作一毛一样。直到后面没有节点之后，将最后一个节点指向头节点（形成环），然后将头节点重新赋值，将最后一个节点的next 的next 置为null（断开环）。

    递归方法：假设后面的N-1个节点已经翻转好了，那么就只需要将第1个节点放到最后就行了，这样一直递归下去，直到没有节点了。最后的一次递归，head（这里其实是原链表的tail） 就是新链表的head。

  11. 快速排序
    两个指针（i,j）一个从前往后，一个从后往前，与数组的第一个数（基准数）进行比较，找到比基准数大（i）或者小(j)的则停止，等到两个指针都找到了，则交换两个数的位置，知道 i>=j，然后将 j 与基准数交换，这样就形成了基准数左边小，右边大的情况。然后递归，将左右两边再进行如此排序。

  13. http和tcp之间的关系

    socket则是对TCP/IP协议的封装和应用(程序员层面上)。
    也可以说，TPC/IP协议是传输层协议，主要解决数据如何在网络中传输，
    而HTTP是应用层协议，主要解决如何包装数据。

    传输层的TCP是基于网络层的IP协议的，而应用层的HTTP协议又是基于传输层的TCP协议的。

    POST把提交的数据则放置在是HTTP包的包体中。
